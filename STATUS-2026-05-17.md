## What this build does

Replaces the dead-end ComfyUI Wan 2.2 Animate approach with a **dedicated Python inference server** that mirrors what Replicate's `wan-2.2-animate-replace` runs internally ÔÇö the official Wan-Video repo's Python API plus Meta's `facebookresearch/sam2`, no ComfyUI involved.

Five rounds of debugging the ComfyUI workflow taught us the actual root cause: ComfyUI's `WanAnimateToVideo` node either doesn't run the required preprocessing pipeline (background-inpainting via SAM2 mask, face cropping, etc.) or expects it pre-built. The Replicate logs (`prediction-t6zqatcyn5rmt0cy6ksbamwddw-logs.txt`) confirmed they bypass ComfyUI and call `wan.WanAnimate.generate(replace_flag=True)` directly after running `sam2.SAM2VideoPredictor.propagate_in_video()` on the source. That's the pipeline this build mirrors.

## Expected first-deploy issues + fixes

These are not bugs ÔÇö they're things I couldn't verify without a running container:

### Issue A ÔÇö `WAN_CONFIGS["animate-14B"]` missing

```
KeyError: 'animate-14B'
```

The config dict key might be different (`animate`, `wan-animate-14B`, etc.). Fix in `server.py`:
```python
from wan.configs import WAN_CONFIGS
print(list(WAN_CONFIGS.keys()))   # exec inside container, find the right key
```

### Issue B ÔÇö `ProcessPipeline` class not found

The class name in `wan.modules.animate.preprocess.process_pipepline` wasn't surfaced by the web search. `_load_model()` falls back to introspection (looks for `ProcessPipeline`, `PreprocessPipeline`, `Pipeline`, then any class with `__call__`). If introspection picks the wrong class, you'll see a runtime error in `progress_step="preprocess"`. Fix:
```python
# inside the container:
from wan.modules.animate.preprocess import process_pipepline
print([n for n in dir(process_pipepline) if not n.startswith('_')])
```

Then hardcode the right class name in `_load_model()` in `server.py`.

### Issue C ÔÇö `WanAnimate.generate()` returns torch tensor not ndarray

`server.py` handles this defensively (calls `.cpu().numpy()` if available). Should Just Work. If imageio chokes on the dtype, the same code block adjusts.

### Issue D ÔÇö Wan model weights missing

If startup fails with "checkpoint not found", verify:
```bash
ls /runpod-volume/ComfyUI/models/diffusion_models/wan2.2_animate_14B_bf16.safetensors
```

If missing, re-run the main app's bootstrap section for Wan weights. The path is already documented in the main README.

### Issue E ÔÇö First request times out

First request takes ~60s extra because the 14B model loads into VRAM. `WAN_ANIMATE_TIMEOUT_SECONDS=600` should cover that. If not, bump it.
